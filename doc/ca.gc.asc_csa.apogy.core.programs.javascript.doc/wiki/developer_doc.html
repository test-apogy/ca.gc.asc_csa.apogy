Introduction

A JavaScriptProgram allows you to invoke Operation Calls on Apogy Variables - and get their attributes - directly from JavaScript.

Each Apogy Variable defined in a Apogy Session is exposed as a global variable in a JavaScriptProgram. For example, if you define the rover variable in Apogy, the global variable rover will be injected into the JavaScriptProgram. Below is a JavaScriptProgram example: 

if (!rover.initialized) {
    rover.init();
}

rover.cmdLinearVelocity(0.1);

var snapshot = rover.centerCamera.takeSnapshot();
java.lang.System.out.println(snapshot.image.width + ", " + snapshot.image.height);

Invoking a JavaScriptProgram

Right-click on the JavaScriptProgram, select Apogy, then Invoke.

Debugging a JavaScriptProgram

  1. Right-click on the JavaScriptProgram, select Apogy, then Debug.
  2. Create a Debug Configuration "Remote JavaScript"
  3. Select the "Mozilla Rhino" connector. Set "localhost" as the "Host" and "9000" as the "Port"
  4. Launch the Debug Configuration

Installation Requirements

  • Eclipse Web Developer Tools (3.7.1+) available from the Eclipse Mars update site.

Plugins

Since an operation on an object must be invoked through the ApogyCoreInvocatorFacade, it cannot be directly exposed in JavaScript. Instead, the object is wrapped in a proxy that automatically calls the ApogyCoreInvocatorFacade when an operation is invoked.

Apogy uses generic proxies for all classes, instead of specific proxies for each metaclass generated from the metamodel.

ca.gc.asc_csa.apogy.core.programs.javascript

Core plugin that defines:

  • JavaScriptProgram Xcore model
  • VariableProxy, TypeMemberProxy and ReferenceProxy
  • ScriptExecutor

JavaScriptProgram (model/apogy_core_javascript_programs.xcore)

A JavaScriptProgram is a Program with a scriptPath attribute. This attribute refers to the file path of the JavaScript file. It is relative to the workspace. For example, if the rover.js file is in the directory scripts of the project MyRover, the scriptPath is MyRover/scripts/rover.js.

ScriptExecutor (src/ca/gc/asc_csa/apogy/core/programs/javascript)

The top-level proxies (those wrapping an Apogy variable) are created at the beginning of the execution of a JavaScriptProgram. For example, if there are two variables in a session, only two proxies are created at the beginning of the execution of the JavaScriptProgram. The proxies for subtypes and features are created on demand as the JavaScriptProgram needs them.

Here is a high-level view of what is happening when you execute a JavaScriptProgram:

  1. ScriptExecutor.execute
    1. Create a new Rhino scope and initialize it with the standard JavaScript objects
    2. Create a proxy for each Apogy variable of the active session
    3. Inject these proxies as global variables into the Rhino scope
    4. Execute the script

If the debug flag is set to true, it starts a RhinoDebugger listening at 127.0.0.1:9000.

Proxy

There are three types of proxies:

  • VariableProxy: wraps an Apogy variable
  • TypeMemberProxy: wraps an Apogy subtype
  • ReferenceProxy: wraps a reference to an EObject

Each proxy extends ScriptableObject. ScriptableObject provides default behavior that makes it easier to expose host objects to JavaScript.

A proxy holds two important attributes:

  • VariableFeatureReference: contains the variable, the subtype (if applicable) and the chain of features the proxy represents
  • JavaScriptProgram: the JavaScriptProgram being executed

When a proxy is created, it dynamically adds properties to itself based on reflection. For example, it will add properties for each operation, feature (attribute and reference) and type member (if applicable). Those properties are not proxies, but instead getters (methods) that generate a proxy when they are accessed from JavaScript. If a property for a feature or type member is never accessed in JavaScript, the underlying proxy is never created. (No proxy are needed for an operation.)

ca.gc.asc_csa.apogy.core.programs.javascript.ui

User interface plugin providing the Execute Script (ScriptExecutorHandler) and Debug Script (ScriptExecutorDebugHandler) context menu items.

ScriptExecutorHandler (asc_csa/apogy/core/programs/javascript/ui/commands)

ScriptExecutorHandler.execute is called when the Execute Script context menu item is clicked. ScriptExecutorHandler.execute gets the JavaScriptProgram and calls ScriptExecutor.execute.

ScriptExecutorDebugHandler (asc_csa/apogy/core/programs/javascript/ui/commands)

ScriptExecutorDebugHandler.execute is called when the Execute Script context menu item is clicked. ScriptExecutorDebugHandler.execute gets the JavaScriptProgram and calls ScriptExecutor.execute with the debug flag set to true.

ca.gc.asc_csa.apogy.core.programs.javascript.tests

Test plugin defining tests for the core plugin.

You can launch the tests from the Run/Debug Configurations | JUnit Plugin Test.

ScriptExecutorTest (src/ca/gc/asc_csa/apogy/core/programs/javascript/tests)

The test case ScriptExecutorTest.executeRoverExampleDemo:

  • loads an Apogy session which contains a JavaScriptProgram
  • calls ScriptExecutor.execute
  • checks that the operation calls have been called through the ApogyCoreInvocatorFacade by querying the data product
